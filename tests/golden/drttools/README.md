# DRTtools reference data

These text exports were generated from the MATLAB DRTtools workflow and are used for diagnostic and numerical golden regression coverage. The pinned DRTtools checkout is commit `034d9c4c4a4916a38a0e2f10381d931ffe1981b3`.

The two top-level `eis_clean_matlab_*` files remain diagnostic comparison fixtures. Matrix-level and end-to-end parity fixtures live in `matrix_piecewise_linear/` and `end_to_end/`.

Use `scripts/run_drttools_reference.m` with a local ignored `DRTtools/` checkout and MATLAB Optimization Toolbox to regenerate reference outputs under `target/matlab_reference/`.

`matrix_piecewise_linear/` was generated by `scripts/export_drttools_matrices.m` using MATLAB R2023b Update 9 and DRTtools commit `034d9c4c4a4916a38a0e2f10381d931ffe1981b3`. The automated test compares `A_re`, `A_im`, `M_1`, `M_2`, quadratic-program `H`/`c`, ridge coefficients, gamma, and reconstructed impedance. Matrix tolerances are `abs <= 1e-10` or `rel <= 1e-8`; coefficients use `rel <= 1e-7`; reconstructed impedance uses `rel <= 1e-8`.

`end_to_end/gaussian_simple_run/` was generated by `scripts/regenerate_matlab_golden.m` using Gaussian RBFs, FWHM coefficient `0.5`, combined real/imaginary data, lambda `1e-3`, first-order regularization, nonnegative coefficients, and inductance fixed to zero. Its automated test compares epsilon, Gaussian kernel matrices, `M_1`, raw RBF coefficients, mapped `gamma(tau)`, objective, and reconstructed impedance.
